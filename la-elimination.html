Section 1.2 Row elimination

¶

You've probably solved small systems of equations by substitution. In order to solve systems with more equations and variables, we take a slightly different point of view and use elimination. The goal of elimination is to transform the system to an equivalent one whose solution(s) we can just read off.

Example 1.2.1.

We start with a system

\begin{align*} x_1 - x_2 - x_3 \amp = 2 \\ 3x_1 - 2x_2 \amp = 9 \\ x_1 - 2x_2 - x_3 \amp = 5 \text{.} \end{align*}

The first step is to use the first equation to eliminate \(x_1\) from the second and third equations. We subtract 3 times equation 1 from equation 2, and 1 times equation 1 from equation 3:

\begin{align*} x_1 - x_2 - x_3 \amp = 2 \amp \amp \\ (3x_1 - 2x_2) - 3(x_1 - x_2 - x_3) \amp = 9 - 3(2)\quad \amp \Rightarrow \quad x_2 +3x_3 \amp = 3\\ (x_1 - 2x_2 - x_3) - 1(x_1 - x_2 - x_3) \amp = 5 - 1(2) \quad \amp \Rightarrow \quad -x_2 \amp = 3\text{.} \end{align*}

I know you want to use that last equation to get rid of \(x_2\) everywhere right away. That would be fine for this particular problem, but we are aiming for a systematic process that works every time.

The next step of the recipe is to leave the first equation alone, and use the second to eliminate \(x_2\) from all the others below it (i.e., the third).

\begin{align*} x_1 - x_2 - x_3 \amp = 2 \amp \amp \\ x_2 + 3x_3 \amp = 3 \amp \amp \\ (-x_2) + (x_2+3x_3) \amp = 3 + 3 \amp \quad \Rightarrow \quad 3x_3 \amp = 6\text{.} \end{align*}

The process in Example 1.2.1 is most commonly known as Gaussian elimination. (It's a misnomer, as the process was known in China thousands of years before Gauss, but never mind.) We could solve the system at the end of the example by starting with the last equation to deduce that \(x_3=2\text{.}\) We then put that value into the second equation and can solve that for \(x_2\text{,}\) etc.

Instead, though, we are going to continue to manipulate the system to get something even simpler.

Example 1.2.2.

(Continuation of Example 1.2.1.) Having reached the last variable and equation, we turn around and eliminate upwards instead:

\begin{align*} (x_1 - x_2 - x_3) + \frac{1}{3}(3x_3) \amp = 2 +\frac{1}{3}(6) \amp \quad \Rightarrow \quad x_1 -x_2 \amp = 4\\ (x_2 + 3x_3) - (3x_3) \amp = 3 - (6)\amp \quad \Rightarrow \quad x_2 \amp = -3\\ 3x_3 \amp = 6 \amp \amp\text{.} \end{align*}

Continue moving back upwards, to the second equation, and use it to eliminate above:

\begin{align*} (x_1 - x_2 ) + (x_2) \amp = 4 + (-3) \amp \quad \Rightarrow \quad x_1 \amp = 1\\ x_2 \amp = -3 \amp \amp \\ 3x_3 \amp = 6 \amp \amp \text{.} \end{align*}

Obviously, the (unique) solution is \(x_1=1\text{,}\) \(x_2=-3\text{,}\) \(x_3=2\text{.}\)

That was a mouthful. We can make it easier if we express the system as \(\bA\bx=\bb\) for a constant matrix and vector, and just manipulate the numbers without carrying around extra notation. We start with the augmented matrix \([\bA,\,\bb]\text{.}\)

Example 1.2.3.

We repeat Example 1.2.1 and Example 1.2.2 but just with the relevant constants. The key is to use zeros as coefficients where there are “silent” variables. The starting augmented matrix is

\begin{equation*} \begin{bmatrix} 1 \amp -1 \amp -1 \amp 2 \\ 3 \amp -2 \amp 0 \amp 9 \\ 1 \amp -2 \amp -1 \amp 5 \end{bmatrix}\text{.} \end{equation*}

First elimination step uses the first row to eliminate below it:

\begin{equation*} \begin{bmatrix} 1 \amp -1 \amp -1 \amp 2 \\ 0 \amp 1 \amp 3 \amp 3 \\ 0 \amp -1 \amp 0 \amp 3 \end{bmatrix}\text{.} \end{equation*}

Next we use the second row to eliminate below it:

\begin{equation*} \begin{bmatrix} 1 \amp -1 \amp -1 \amp 2 \\ 0 \amp 1 \amp 3 \amp 3 \\ 0 \amp 0 \amp 3 \amp 6 \end{bmatrix}\text{.} \end{equation*}

Having reached the last row, we turn around and eliminate above it.

\begin{equation*} \begin{bmatrix} 1 \amp -1 \amp 0 \amp 4 \\ 0 \amp 1 \amp 0 \amp -3 \\ 0 \amp 0 \amp 3 \amp 6 \end{bmatrix}\text{.} \end{equation*}

We move up to the second row and eliminate above that:

\begin{equation*} \begin{bmatrix} 1 \amp 0 \amp 0 \amp 1 \\ 0 \amp 1 \amp 0 \amp -3 \\ 0 \amp 0 \amp 3 \amp 6 \end{bmatrix}\text{.} \end{equation*}

Finally, to be super pedantic, we normalize each row:

\begin{equation*} \begin{bmatrix} 1 \amp 0 \amp 0 \amp 1 \\ 0 \amp 1 \amp 0 \amp -3 \\ 0 \amp 0 \amp 1 \amp 2 \end{bmatrix}\text{.} \end{equation*}

The last column is now the solution.

Before we generalize, let's look at an example that works out differently.

Example 1.2.5.

Let's solve the system having augmented matrix

\begin{equation*} \begin{bmatrix} 1 \amp 1 \amp -1 \amp 4 \\ 1 \amp 2 \amp 2 \amp 3 \\ 2 \amp 1 \amp -5 \amp 9 \end{bmatrix}\text{.} \end{equation*}

Use multiples of the first row to eliminate below it.

\begin{equation*} \begin{bmatrix} 1 \amp 1 \amp -1 \amp 4 \\ 0 \amp 1 \amp 3 \amp -1 \\ 0 \amp -1 \amp -3 \amp 1 \end{bmatrix} \end{equation*}

Use a multiple of the second row to eliminate below that.

\begin{equation*} \begin{bmatrix} 1 \amp 1 \amp -1 \amp 4 \\ 0 \amp 1 \amp 3 \amp -1 \\ 0 \amp 0 \amp 0 \amp 0 \end{bmatrix} \end{equation*}

We have reached the last row, and it's time to turn around and eliminate upwards. But we can't, because the last row is entirely zero. So skip that and move up to the next row, eliminating upwards again in the second column:

\begin{equation*} \begin{bmatrix} 1 \amp 0 \amp -1 \amp 5 \\ 0 \amp 1 \amp 3 \amp -1 \\ 0 \amp 0 \amp 0 \amp 0 \end{bmatrix}\text{.} \end{equation*}

This is as simple as we can get things. The last row is telling us the astounding fact that \(0=0\text{,}\) i.e., nothing at all. The fact that there was nothing there to determine \(x_3\) means that we can set it to be anything we like, say \(x_3=s\) for arbitrary \(s\text{.}\) Now the second row says

\begin{equation*} x_2 = -1 - 3s\text{,} \end{equation*}

which is as specific as we can get. Finally, the first row says

\begin{equation*} x_1 = 5 + s\text{.} \end{equation*}

So \([5+s,-1-3s,s]\) is a solution for every \(s\text{,}\) and we have infinitely many solutions.

The process demonstrated in Example 1.2.5 is best known as Gauss-Jordan elimination, though I prefer to call it row elimination. As seen in the examples, row elimination consists of two phases, one downward (Gaussian elimination) and one upward. The goal is to put the augmented matrix into a special form.

Time for some more formal definitions. We start with the downward phase of the elimination. It's organized a tad differently in order to make everything easier to describe. In what follows we use the term leading nonzero to mean the first (leftmost) entry of a row that is not zero.

Definition 1.2.6. Row elimination Phase 1 (Gaussian elimination).
  1. Set \(i=1\text{.}\)
  2. Find the leftmost leading nonzero in rows \(i\) and below. The column of this leading nonzero is known as the pivot column. If no such column exists, stop.
  3. As necessary, swap rows and/or multiply a row by a constant to put a 1 in the pivot column of row \(i\text{.}\)
  4. Add multiples of row \(i\) to the rows below it in order to put zeros into the pivot column.
  5. Increment \(i\) and return to step 2.

At the end of Phase 1 the augmented matrix has a simple form. However, it's not the cleanest form to work with theoretically, so we continue.

Definition 1.2.7. Row elimination Phase 2.
  1. Set \(i=m\) (the number of equations).
  2. Use multiples of row \(i\) to put zeros above the leading 1 in that row.
  3. Decrement \(i\text{.}\) If \(i\gt 1\text{,}\) return to step 2.

At the end of Phase 2, the matrix is in a form that we define formally here.

Definition 1.2.10. RREF.

A matrix is in RREF (reduced row echelon form) if it meets all of these requirements:

  1. Any rows of all zeros appear below all nonzero rows.
  2. The leading nonzero of any row is a one.
  3. Every leading one is the only nonzero in its column.

The columns that have leading ones are called pivot columns. The other columns are called free columns.

As seen in the examples above, the RREF of an augmented matrix represents a linear system that we can solve by inspection:

  • Ignore all zero rows.
  • If a leading one occurs in the last column, the system is inconsistent.
  • Otherwise, each variable associated with a free column is assigned to a free parameter (e.g., \(s\text{,}\) \(t\text{,}\) etc.).
  • Use the pivot columns to solve for their corresponding variables in terms of the free parameters.

MATLAB has a command rref to put a matrix in RREF. However, it's not ideal, because MATLAB uses floating point and cannot always produce exact zeros. You can also try Wolfram Alpha for this.
